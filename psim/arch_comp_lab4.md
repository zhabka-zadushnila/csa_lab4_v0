# Вариант

`alg | acc | neum | mc | tick | binary | stream | mem | cstr | prob1 | cache`

`alg` -- синтаксис языка напоминает C/Rust. Поддерживаются математические выражения с учётом приоритетов операторов.

`acc` -- система команд построена вокруг аккумулятора (AC). Все арифметические и логические операции изменяют AC. Ввод-вывод осуществляется через AC.

`neum` -- фон Неймановская архитектура (единая память для кода и данных).

`mc` -- микропрограммное управление. Модель процессора исполняет микрокод, сохранённый в отдельной памяти микропрограмм.

`tick` -- моделирование выполняется с точностью до такта.

`stream` -- ввод-вывод осуществляется как поток токенов. При старте модели предзагружается буфер ввода. При обращении к порту ввода процессор получает токен. При записи в порт вывода токен добавляется в буфер вывода.

`mem` -- порты ввода-вывода отображаются в память (MMIO) и доступны штатными командами LD/ST.

`cstr` -- строки хранятся в памяти как нуль-терминированные последовательности (C-строки), одно 32-битное слово на символ.

`cache` -- работа с памятью реализуется через кеш L1 (8 КБ, 128 строк по 64 байта). Доступ к кешу -- 1 такт, к основной памяти -- 10 тактов.

# Язык AlсoScript

## Примеры программ

```algo
fn main() -> i32 {
    cout << ((14 + 34 * 12)/22 + 34)*242 * (15+4); 
    return 1;
}
```

```algo
fn main() -> i32 {
    cout << |3 * -15 + 2|; 
    return 1;
}
```

```algo
fn main() -> i32 {
    i32 a = 0;
    i32 b = 1;
    for (i32 i = 0; i < 10; i = i + 1) {
        i32 next = a + b;
        cout << next;
        a = b;
        b = next;
    }
    return 1;
}
```

```algo
fn main() -> i32 {
    ptr a = [8, 4, 'a', [1, 2, 3]];
    ptr b[15];

    for (i32 i = 0; i < 13; i++) {
        b[i] = i;
    }

    for (i32 i = 0; i < 4; i++) {
        if (i != 3) {
            cout << a[i];
        } else {
            i32 k = 3;
            while (k != 0) {
                k--;
                cout << a[i][k] << '\n';
            }
        }
    }
    return 1;
}
```

```algo
fn main() -> i32 {
    i32 x;
    cin >> x;
    cout << x;
    return 1;
}
```

## Типы

- `i32` -- знаковое 32-битное целое (базовый тип)
- `ptr` -- указатель на массив 32-битных слов или нуль-терминированную строку

## Синтаксис (EBNF)

```ebnf
<program>      ::= <declaration>*

<declaration>  ::= <func_decl>

<func_decl>    ::= "fn" <identifier> "(" <param_list> ")" [ "->" <type> ] "{" <statement_list> "}"
<param_list>   ::= [ <param_decl> ("," <param_decl>)* ]
<param_decl>   ::= <type> <identifier>

<statement>    ::= <var_decl> ";"
                 | <assignment> ";"
                 | <if_stmt>
                 | <while_stmt>
                 | <for_stmt>
                 | "return" <expression> ";"
                 | "cout" ("<<" <expression>)+ ";"
                 | "cin" (">>" <identifier>)+ ";"
                 | <block>

<var_decl>     ::= <type> <identifier> [ "=" <expression> ]
                 | <type> <identifier> "[" <expression> "]"

<type>         ::= "i32" | "ptr"

<block>        ::= "{" <statement_list> "}"

<if_stmt>      ::= "if" <expression> <block> [ "else" <block> ]
<while_stmt>   ::= "while" <expression> <block>
<for_stmt>     ::= "for" "(" <statement> <expression> ";" <expression> ")" <block>

<expression>   ::= <assignment_expr>

<assignment_expr> ::= <logical_or_expr> ( "=" <assignment_expr> )*

<logical_or_expr>  ::= <logical_and_expr> ( "||" <logical_and_expr> )*
<logical_and_expr> ::= <bitwise_or_expr> ( "&&" <bitwise_or_expr> )*
<bitwise_or_expr>  ::= <bitwise_xor_expr> ( "|" <bitwise_xor_expr> )*
<bitwise_xor_expr> ::= <bitwise_and_expr> ( "^" <bitwise_and_expr> )*
<bitwise_and_expr> ::= <equality_expr> ( "&" <equality_expr> )*
<equality_expr>    ::= <relational_expr> ( ("==" | "!=") <relational_expr> )*
<relational_expr>  ::= <shift_expr> ( ("<" | "<=" | ">" | ">=") <shift_expr> )*
<shift_expr>       ::= <additive_expr> ( ("<<" | ">>") <additive_expr> )*
<additive_expr>    ::= <multiplicative_expr> ( ("+" | "-") <multiplicative_expr> )*
<multiplicative_expr> ::= <unary_expr> ( ("*" | "/" | "%") <unary_expr> )*

<unary_expr>   ::= "-" <unary_expr>
                 | "!" <unary_expr>
                 | <postfix_expr>

<postfix_expr> ::= <primary> ( "[" <expression> "]"
                               | "(" <arg_list> ")"
                               | "++" | "--" )*

<primary>      ::= <int_literal>
                 | <char_literal>
                 | <string_literal>
                 | <bool_literal>
                 | <identifier>
                 | "|" <expression> "|"
                 | "(" <expression> ")"
                 | "[" <array_elements> "]"

<arg_list>     ::= [ <expression> ("," <expression>)* ]
<array_elements> ::= <expression> ("," <expression>)*

<identifier>   ::= [a-zA-Z_] [a-zA-Z0-9_]*
<int_literal>  ::= <decimal> | "0x" <hex> | "0b" <binary>
<char_literal> ::= "'" <char> "'"    (* поддерживаются escape-последовательности \n, \t, \\, \' *)
<string_literal> ::= '"' <string> '"' (* поддерживаются escape-последовательности \n, \t, \\, \" *)
<bool_literal> ::= "true" | "false"
```

## Семантика

**Стратегия вычислений:** Жадная (eager evaluation). Аргументы функций вычисляются до вызова. Выражения вычисляются слева направо согласно приоритетам операторов (Pratt-парсер). Логические операторы `&&` и `||` используют короткое замыкание (short-circuit).

**Передача аргументов:** Call-by-value для `i32`. Для `ptr` передаётся адрес (call-by-reference).

**Области видимости:** Лексическая. Глобальные имена (функции) видны везде. Локальные переменные существуют в пределах блока `{ ... }`. Допускается затенение (shadowing).

**Типизация:** Статическая, строгая. Типы известны на этапе компиляции, неявные приведения запрещены.

**Литералы:**
- Целые числа: десятичные (`42`), шестнадцатеричные (`0x2A`), двоичные (`0b101010`)
- Символы: `'a'`, `'\n'` -- преобразуются в ASCII-код (i32)
- Строки: `"hello"` -- размещаются в секции данных как null-terminated последовательность 32-битных слов
- Булевы: `true` (1), `false` (0)

# Компилятор

## Устройство

Компилятор реализован на Rust и состоит из четырёх модулей:

1. **Лексический анализатор** (`lexer.rs`) -- разбивает исходный текст на поток токенов
2. **Синтаксический анализатор** (`parser.rs`) -- строит AST методом Pratt-парсинга
3. **Оптимизатор** (`optimize_tree`) -- выполняет свёртку констант
4. **Генератор кода** (`codegen.rs`) -- двухпроходный: сбор символов, затем генерация инструкций с отложенными фиксапами

## CLI

```
compiler <input.algo> <output.bin> [--listing listing.txt]
```

Выходной файл -- плоский бинарник без заголовка: сначала секция кода, затем данные, затем BSS (дополненный нулями). Каждое 32-битное слово записывается в little-endian порядке байтов.

## Пример трансляции

Исходный код (`prog2.algo`):
```algo
fn main() -> i32 {
    i32 a = 3;
    i32 b = -15;
    cout << |a * b + 2|;
    return 1;
}
```

AST после парсинга:
```
Block
├── FuncDecl "main"
│   ├── args: []
│   ├── ret_type: Some("i32")
│   └── body: Block
│       ├── VarDecl "a" = Integer(3)
│       ├── VarDecl "b" = Neg(Integer(15))
│       ├── Cout
│       │   └── Absolute
│       │       └── Operation(Add)
│       │           ├── Operation(Mul)
│       │           │   ├── Atom(Variable("a"))
│       │           │   └── Atom(Variable("b"))
│       │           └── Integer(2)
│       └── Return(Integer(1))
```

Полученный машинный код (листинг):
```
0x0000  fn_main:
0x0000  0x10200003  LD #3
0x0001  0x1140000C  ST [0x0000C]
0x0002  0x1020000F  LD #15
0x0003  0x04000000  INV
0x0004  0x2340000C  MUL [0x0000C]
0x0005  0x20200002  ADD #2
0x0006  0x07000000  ABS
0x0007  0x11400004  ST [0x00004]
0x0008  0x10200001  LD #1
0x0009  0x0A000000  RET
0x000A  0x0A000000  RET
0x000B  0x01000000  HLT
```

Карта памяти (адреса):
- `0x0000..0x000B`: код (12 инструкций)
- `0x000C`: переменная `var_a` (BSS, 1 слово)
- `0x000D`: переменная `var_b` (BSS, 1 слово)
- `0x0004`: порт вывода `OUT_PORT` (MMIO)

## Оптимизатор

Выполняет свёртку констант на уровне AST. Выражения вида `Operation(Add, [Integer(2), Integer(3)])` заменяются на `Integer(5)`. Рекурсивно обходит все узлы дерева.

Пример: `((14 + 34 * 12) / 22 + 34) * 242 * (15 + 4)` = `243694` -- компилируется в единственную инструкцию:
```
0x0000  0x1023B7EE  LD #243694
0x0001  0x11400004  ST [0x00004]
0x0002  0x10200001  LD #1
0x0003  0x0A000000  RET
```

# Система команд

## Кодирование инструкций

32-битная инструкция кодируется как:

```
Биты [31:24] -- опкод (8 бит)
Биты [23:21] -- режим адресации (3 бита)
Биты [20:0]  -- операнд / адрес (21 бит)
```

Функция кодирования: `(opcode << 24) | (mode << 21) | (operand & 0x1FFFFF)`

## Режимы адресации

| Код | Название | Синтаксис | Описание |
| :-: | :------- | :-------- | :------- |
| 000 | Безадресный | `OP` | Операнд игнорируется |
| 001 | Непосредственный | `OP #value` | Операнд -- значение |
| 010 | Косвенный | `OP [addr]` | Операнд -- адрес ячейки, содержащей целевой адрес |
| 011 | Косвенный по стэку | `OP [SP+ofs]` | SP+ofs -- адрес ячейки, содержащей целевой адрес |
| 100 | Автоинкрементный | `OP (addr)+` | Операнд -- адрес ячейки с целевым адресом; после исполнения addr++ |
| 101 | Автодекрементный | `OP -(addr)` | Перед исполнением addr--; затем addr -- адрес ячейки с целевым адресом |

## Таблица инструкций

### Безадресные (mode = 000)

| Инструкция | Опкод | Описание |
| :--------- | :---: | :------- |
| `NOP` | 0x00 | Нет операции |
| `HLT` | 0x01 | Останов симуляции |
| `CLA` | 0x02 | AC = 0 |
| `CMA` | 0x03 | AC = ~AC |
| `INV` | 0x04 | AC = ~AC + 1 (дополнительный код) |
| `INC` | 0x05 | AC = AC + 1 |
| `DEC` | 0x06 | AC = AC - 1 |
| `ABS` | 0x07 | AC = \|AC\| |
| `PUSH` | 0x08 | \[SP\] = AC, SP-- |
| `POP` | 0x09 | SP++, AC = \[SP\] |
| `RET` | 0x0A | IP = \[SP\], SP++ (возврат из подпрограммы) |
| `EXT8` | 0x0B | Знаковое расширение младшего байта AC до 32 бит |
| `EXT16` | 0x0C | Знаковое расширение младшего полуслова AC до 32 бит |

### Память (mode: immediate/direct/indirect)

| Инструкция | Опкод | Описание |
| :--------- | :---: | :------- |
| `LD` | 0x10 | AC = operand |
| `ST` | 0x11 | \[addr\] = AC (только прямой/косвенный режимы) |

### Арифметика и логика (mode: immediate/direct)

| Инструкция | Опкод | Описание |
| :--------- | :---: | :------- |
| `ADD` | 0x20 | AC = AC + operand |
| `ADC` | 0x21 | AC = AC + operand + CF |
| `SUB` | 0x22 | AC = AC - operand |
| `MUL` | 0x23 | AC = AC * operand |
| `DIV` | 0x24 | AC = AC / operand |
| `MOD` | 0x25 | AC = AC % operand |
| `CMP` | 0x26 | AC - operand (устанавливает флаги Z, N) |
| `AND` | 0x27 | AC = AC & operand |
| `OR` | 0x28 | AC = AC \| operand |
| `XOR` | 0x29 | AC = AC ^ operand |

### Переходы (mode: direct)

| Инструкция | Опкод | Описание |
| :--------- | :---: | :------- |
| `JMP` | 0x30 | IP = addr |
| `JEQ` | 0x31 | IP = addr, если Z = 1 |
| `JNE` | 0x32 | IP = addr, если Z = 0 |
| `JLT` | 0x33 | IP = addr, если N = 1 |
| `JGE` | 0x34 | IP = addr, если N = 0 |
| `JCS` | 0x35 | IP = addr, если C = 1 |
| `JCC` | 0x36 | IP = addr, если C = 0 |
| `JVS` | 0x37 | IP = addr, если V = 1 |
| `JVC` | 0x38 | IP = addr, если V = 0 |
| `CALL` | 0x39 | \[SP\] = IP, SP--, IP = addr |

# Компиляция конструкций языка

## Выражения

Компилятор использует аккумуляторную модель: результат каждого подвыражения помещается в AC.

### Литералы

| Конструкция | Генерируемый код |
| :---------- | :--------------- |
| `42` | `LD #42` |
| `'a'` | `LD #97` |
| `true` | `LD #1` |
| `"hello"` | `LD #@str_<offset>` (адрес строки в data, через фиксап) |

### Арифметика

Левый операнд вычисляется в AC. Правый операнд подаётся через арифметическую инструкцию:

- `a + b` → `LD a` / `ADD b`
- `a - b` → `LD a` / `SUB b`
- `a * b` → `LD a` / `MUL b`
- `a / b` → `LD a` / `DIV b`
- `a % b` → `LD a` / `MOD b`

Если правый операнд -- сложное выражение, значение левого операнда сохраняется во временную переменную.

### Сравнения

`a == b`, `a != b`, `a < b`, `a <= b`, `a > b`, `a >= b` -- компилируются через `CMP` + условный переход:

```
    LD <левый операнд>
    CMP <правый операнд>
    Jcc true_label
    JMP false_label
```

Результат сравнения (1 или 0) остаётся в AC.

### Логические операторы

`&&` и `||` -- с коротким замыканием (short-circuit). Внутри условий (if/while/for) генерируют переходы сразу на соответствующие метки. В позиции выражения -- дают 1 или 0 через условные переходы.

### Отрицание и not

`-expr` → `INV` (AC = ~AC + 1)

`!expr` → `CMP #0` + условный переход (1 если AC == 0, иначе 0)

### Модуль

`\|expr\|` → `ABS`

## Переменные и присваивание

### Объявление

```
i32 a = 42;
```
→ вычислить 42 → `ST a`

Без инициализатора переменная обнуляется на этапе загрузки (BSS).

### Присваивание

```
a = expr;
```
→ `LD expr` → `ST a`

### Инкремент / декремент

```
a++;
a--;
```
→ `LD a` → `INC`/`DEC` → `ST a`

## Управляющие конструкции

### if/else

```
if (cond) { ... } else { ... }
```
→
```
    <compile_condition cond -> true_label, false_label>
true_label:
    <then_body>
    JMP end_label
false_label:
    <else_body>
end_label:
```

### while

```
while (cond) { ... }
```
→
```
loop_label:
    <compile_condition cond -> body_label, end_label>
body_label:
    <body>
    JMP loop_label
end_label:
```

### for

```
for (init; cond; step) { ... }
```
→
```
    <init>
loop_label:
    <compile_condition cond -> body_label, end_label>
body_label:
    <body>
    <step>
    JMP loop_label
end_label:
```

## Ввод-вывод

### cout

```
cout << expr1 << expr2;
```
→ для каждого выражения: `LD expr` → `ST [0x0004]` (OUT_PORT)

### cin

```
cin >> x;
```
→ `LD [0x0000]` (IN_PORT) → `ST x`

Порты ввода-вывода memory-mapped:
- `IN_PORT` = `0x0000` -- чтение токена из потока ввода
- `OUT_PORT` = `0x0004` -- запись токена в поток вывода

## Функции

### Вызов

Аргументы передаются справа налево через стек. Возвращаемое значение -- в AC.

```
result = fib(n);
```
→
```
    LD n
    PUSH
    CALL fn_fib
    POP          ; очистка стека от аргумента
```

### Объявление

Параметры функции размещаются в BSS как глобальные переменные (рекурсия не поддерживается -- все переменные статические).

```
fn fib(i32 n) -> i32 { ... }
```
→
```
fn_fib:
    <body>
    RET
```

## Массивы и указатели

### Объявление

```
ptr a[10];         -- массив на 10 элементов (BSS)
ptr a = [1,2,3];   -- массив, инициализированный в data
```

### Индексация

Чтение `a[i]`:
```
    LD #array_base   ; загружаем базовый адрес
    ST __tmp         ; сохраняем во временную
    LD i             ; индекс
    ADD __tmp        ; AC = базовый адрес + индекс
    ST __tmp         ; сохраняем эффективный адрес
    LD [__tmp]       ; косвенная загрузка
```

Запись `a[i] = val`:
```
    LD #array_base
    ST __tmp
    LD i
    ADD __tmp
    ST __tmp
    LD val
    ST [__tmp]       ; косвенная запись
```

# Процессор

## Характеристики

- 32-битный аккумуляторный процессор
- Фон Неймановская архитектура
- Адресное пространство: 128 МБ (4194304 32-битных слов)
- Кеш L1: 8 КБ (2048 слов), direct-mapped, 128 строк × 64 байта, разбиение адреса: tag=19 бит, index=7 бит, offset=6 бит
- Микропрограммное управление

## Регистры

| Регистр | Название | Разрядность | Назначение |
| :------ | :------- | :---------- | :--------- |
| AC | Accumulator | 32 бита | Аккумулятор, основной рабочий регистр |
| DR | Data Register | 32 бита | Буфер данных для взаимодействия с памятью |
| CR | Command Register | 32 бита | Хранит текущую выполняемую инструкцию |
| AR | Address Register | 32 бита | Регистр адреса памяти |
| IP | Instruction Pointer | 32 бита | Указатель текущей инструкции |
| SP | Stack Pointer | 32 бита | Указатель стека (растёт вниз) |
| PS | Program Status | 5 бит | Флаги: Inp, Z, N, C, V |

### Флаги PS

| Бит | Флаг | Описание |
| :-: | :--- | :------- |
| 4 | Inp | Доступны данные на порту ввода |
| 3 | Z | Результат последней операции равен нулю |
| 2 | N | Результат последней операции отрицателен |
| 1 | C | Перенос (unsigned overflow) |
| 0 | V | Переполнение (signed overflow) |

     │       │       │       │       │       │          │
     └───────┴───────┴───────┴───────┴───────┘          │
                         │                              │
                      ┌──▼──┐                           │
                      │ ALU │◄──────────────────────────┘
                      └─────┘

### Структура кеша

Кеш L1 — direct-mapped, 128 строк (sets) по 64 байта (16 слов). Адрес (32 бита) разбивается на три поля:

```
Биты [31:13] — tag  (19 бит)
Биты [12:6]  — index (7 бит)
Биты [5:0]   — offset (6 бит)
```

- **tag** (19 бит): старшая часть адреса, сравнивается с тегом кеш-строки
- **index** (7 бит): номер строки кеша (0–127)
- **offset** (6 бит): смещение внутри строки (0–63 байта)

Попадание в кеш — 1 такт, промах — обращение к основной памяти (10 тактов) с заполнением кеш-строки.

## Микропрограммное управление

```
[ CR (Opcode + Mode) ]
         |
         v
[ Mapping ROM ]  ──→  (opcode → microcode start address)
         |
         v      ┌───────┐
[ MPC ]─────────┤ PS fl │
         │      └───────┘
         v
[ Microcode ROM ]
         |
         v
[ MIR ]
    |         |
    |         +──→ [Control Signals]
    |               - AC_in, AC_out, DR_in, DR_out, AR_in
    |               - ALU_Opcode, Mem_Read, Mem_Write
    +──→ [Next Address Logic]
          - Increment (default)
          - Jump (unconditional)
          - Branch (conditional on PS flags)
```
### 1. Формат микрокоманды (Распределение 40 бит)

Ширина в 40 бит позволяет управлять всеми узлами процессора параллельно за один такт.

| Группа сигналов | Биты | Описание |
| :--- | :---: | :--- |
| **Источники шины (OUT)** | **[39:34] (6 бит)** | Открытие выходов регистров на Internal Data Bus (каждый бит — отдельный тристабильный буфер). *Одновременно должен быть выставлен только 1 бит.* |
| **Приемники шины (IN)** | **[33:27] (7 бит)** | Сигналы записи (Write Enable) из шины в регистры. Можно выставлять несколько бит параллельно. |
| **Счетчики (CNT)** | **[26:24] (3 бита)** | Инкремент/декремент регистров-счетчиков. |
| **Память и Кэш (MEM)** | **[23:21] (3 бита)** | Управление системной шиной памяти. |
| **АЛУ (ALU)** | **[20:16] (5 бит)** | Выбор выполняемой операции в АЛУ. |
| **Секвенсор (SEQ_CTRL)** | **[15:14] (2 бита)** | Управление счетчиком микрокоманд (MPC). |
| **Условия (SEQ_COND)** | **[13:10] (4 бита)** | Выбор флага для условного ветвления микрокода. |
| **Адрес (NEXT_ADDR)** | **[9:0] (10 бит)** | Адрес следующей микрокоманды (адресное пространство микрокода — 1024 слова). |

---

### 2. Детальная расшифровка полей

#### Источники данных для внутренней шины (Биты 39–34)
Исключительно горизонтальный подход: 1 бит = 1 выход на шину.
*   **39: `AC_out`** — выдать значение Аккумулятора на шину.
*   **38: `DR_out`** — выдать значение Регистра Данных на шину.
*   **37: `IP_out`** — выдать Указатель Инструкций.
*   **36: `SP_out`** — выдать Указатель Стека.
*   **35: `CR_imm_out`** — выдать младшие 21 бит из CR (операнд), расширенные до 32 бит, на шину.
*   **34: `ALU_out`** — выдать результат вычисления АЛУ на шину.

#### Приемники данных с внутренней шины (Биты 33–27)
*   **33: `AC_in`** — защелкнуть данные с шины в Аккумулятор.
*   **32: `DR_in`** — защелкнуть данные в Регистр Данных (для записи в память).
*   **31: `AR_in`** — защелкнуть данные в Регистр Адреса (для обращения к памяти/портам).
*   **30: `IP_in`** — запись в Указатель Инструкций (переходы).
*   **29: `SP_in`** — запись в Указатель Стека.
*   **28: `CR_in`** — запись в Регистр Команд (на этапе Fetch).
*   **27: `PS_in`** — обновить флаги (Z, N, C, V) по текущему состоянию АЛУ.

#### Инкременты и Декременты (Биты 26–24)
*   **26: `IP_inc`** — `IP = IP + 1`.
*   **25: `SP_inc`** — `SP = SP + 1` (POP / возврат из подпрограммы).
*   **24: `SP_dec`** — `SP = SP - 1` (PUSH / вызов функции).

#### Работа с памятью (L1 Cache / MMIO) (Биты 23–21)
*   **23: `Mem_Read`** — инициировать чтение по адресу в `AR`. Результат попадет в `DR`.
*   **22: `Mem_Write`** — инициировать запись значения из `DR` по адресу в `AR`.
*   **21: `Mem_Wait`** — **Ключевой бит для `cache` и `tick`**. Если выставлен, микросеквенсор останавливается (stall) до тех пор, пока контроллер памяти не выдаст сигнал Ready (1 такт при попадании в кэш, 10 тактов при промахе). При обращении к `0x0000` (порт ввода), процессор будет ждать токен из `stream`, если буфер пуст.

#### Управление АЛУ (Биты 20–16)
5 бит позволяют закодировать 32 операции. АЛУ жестко подключено к шине и Аккумулятору (левый операнд всегда AC, правый — значение на шине).
*   `00000`: PASS_BUS (пропустить значение с шины сквозь АЛУ)
*   `00001`: ADD
*   `00010`: SUB
*   `00011`: MUL
*   ... и так далее для всех целочисленных операций, включая `CMP`, `INV`, `ABS`.

#### Управление секвенсором (Биты 15–0)
*   **[15:14] SEQ_CTRL**:
    *   `00`: `NEXT` (переход к MPC + 1, поле NEXT_ADDR игнорируется).
    *   `01`: `JUMP` (если условие истинно, MPC = NEXT_ADDR, иначе MPC + 1).
    *   `10`: `MAP` (загрузить в MPC адрес микропрограммы из Mapping ROM на основе опкода в `CR`).
    *   `11`: Резерв.
*   **[13:10] SEQ_COND** (Мультиплексор условий ветвления):
    *   `0000`: Всегда истинно (`Unconditional`).
    *   `0001`: Флаг `Z == 1` (Zero).
    *   `0010`: Флаг `N == 1` (Negative).
    *   `0011`: Флаг `C == 1` (Carry).
    *   `0100`: Флаг `V == 1` (Overflow).
    *   `0101`: Флаг `Inp == 1` (Есть данные в порту).
    *   `1000` - `1111`: Проверки режима адресации (Mode из `CR` [23:21]) для маршрутизации микрокода (Direct, Indirect, Auto-inc и т.д.).
*   **[9:0] NEXT_ADDR**: 10-битный адрес перехода (поддержка до 1024 микрокоманд, что с запасом хватит для косвенной адресации и других операций).
*

### 3. Типы микрокоманд

Поскольку формат строго горизонтальный, деления на "типы" на уровне архитектуры нет (каждая строка — это всегда 40 бит). Однако *логически* микрокоманды делятся на следующие фазы:

#### 1. Фаза выборки инструкции (Fetch Cycle)
Выполняется перед каждой инструкцией.
*   **Такт 1:** `IP_out`, `AR_in` *(копируем IP в регистр адреса)*.
*   **Такт 2:** `Mem_Read`, `Mem_Wait` *(ждем ответа от L1 Cache)*.
*   **Такт 3:** `DR_out`, `CR_in`, `IP_inc` *(инструкция загружена, сдвигаем IP)*.
*   **Такт 4:** `SEQ_CTRL = MAP` *(переход к микрокоду конкретной команды)*.

#### 2. Фаза вычисления эффективного адреса (Address Resolution)
Для сложных режимов адресации (Mode).
*   *Пример (Indirect):* `CR_imm_out`, `AR_in` -> `Mem_Read`, `Mem_Wait` -> `DR_out`, `AR_in`.

#### 3. Фаза исполнения АЛУ (Execute - Арифметика)
Для команд вроде `ADD #14`.
*   **Такт 1:** `CR_imm_out` (операнд на шину), `ALU_OP = ADD`, `ALU_out`, `AC_in`, `PS_in`.
*   **Такт 2:** `JUMP` на микропрограмму Fetch.

#### 4. Фаза памяти (I/O и переменные)
Для команд вроде `ST [0x0004]` (Вывод в OUT_PORT через MMIO).
*   **Такт 1:** `CR_imm_out`, `AR_in` *(адрес порта `0x0004` в AR)*.
*   **Такт 2:** `AC_out`, `DR_in` *(данные для записи в буфер данных)*.
*   **Такт 3:** `Mem_Write`, `Mem_Wait` *(запись в Stream Out)*.
*   **Такт 4:** Переход на Fetch.
